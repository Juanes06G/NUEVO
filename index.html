¿ QUE ES UN CONTROL DE VERSIONES ?

"Un control de versiones es un sistema que permite gestionar y registrar los cambios realizados en archivos y directorios a lo largo del tiempo. Es fundamental en el desarrollo de software para mantener un historial de modificaciones, coordinar el trabajo entre varios desarrolladores y revertir a versiones anteriores si es necesario"


¿QUE ES GIT?

"Git es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar y rastrear cambios en el código fuente de un proyecto. A diferencia de los sistemas de control de versiones centralizados, Git permite a cada desarrollador tener una copia completa del repositorio en su propia máquina, lo que facilita el trabajo en equipos distribuidos y la gestión de cambios."

¿ QUE ES UN REPOSITORIO?

"Un repositorio es un lugar donde se almacena y organiza el código fuente y otros archivos relacionados con un proyecto. Existen dos tipos principales de repositorios:"


LOCALES

-Son repositorios que existen en la máquina del usuario. Cada usuario puede tener su propio repositorio local, donde puede realizar cambios y hacer pruebas de manera independiente.

REMOTOS

-Son repositorios que están alojados en servidores en la red (como GitHub, GitLab, Bitbucket). Los repositorios remotos sirven para compartir cambios entre varios usuarios y equipos.

¿QUE ES GITHUB?

"GitHub es una plataforma de alojamiento de código fuente basada en la web que utiliza Git como sistema de control de versiones. Permite a los desarrolladores almacenar sus repositorios en la nube, colaborar en proyectos, realizar revisiones de código y gestionar problemas y solicitudes de cambio. Ofrece funcionalidades adicionales como integración continua, documentación y herramientas para gestionar proyectos."

¿ CUALES SON LOS ESTADOS EN GIT?

"En Git, los archivos pueden estar en uno de tres estados:"


MODIFICADO

El archivo ha sido cambiado, pero esos cambios aún no se han registrado en el sistema de control de versiones.

PREPARADO (STAGED)

El archivo ha sido marcado para incluirse en el próximo commit, pero aún no se ha registrado.

CONFIRMADO (COMMITTED)

El archivo ha sido registrado en el repositorio y los cambios han sido almacenados en el historial de commits

¿QUE ES UNA RAMA EN GIT Y PARA QUE SIRVE?

"Una rama en Git es una línea independiente de desarrollo que permite trabajar en diferentes versiones del proyecto simultáneamente. Las ramas se utilizan para:"

- Desarrollar nuevas características sin afectar la rama principal (generalmente llamada main o master).
- Corregir errores sin interrumpir el desarrollo principal.
- Realizar experimentos y pruebas de manera aislada."

Función: Crea un nuevo directorio con el nombre especificado en < nombre >.

¿QUE ES UNA RAMA EN GIT Y PARA QUE SIRVE?

"Una rama en Git es una línea independiente de desarrollo que permite trabajar en diferentes versiones del proyecto simultáneamente. Las ramas se utilizan para:"

*GIT INIT*

Función: Inicializa un nuevo repositorio Git vacío en el directorio actual. Crea un subdirectorio llamado .git donde se almacenan todos los archivos necesarios para el repositorio.

*GIT CLONE*< url >

Función: Clona un repositorio remoto en tu máquina local. Copia todos los archivos, el historial de cambios y las ramas del repositorio remoto a un nuevo directorio local.

*GIT ADD* < archivo >

Función: Añade cambios en el archivo especificado al área de preparación (staging area) para el próximo commit. Puedes usar git add . para añadir todos los archivos modificados en el directorio actual.

*GIT COMMIT-M* "< mensaje >

Función: Registra los cambios en el repositorio con un mensaje descriptivo. El mensaje debe entre comillas y describe el propósito de los cambios realizados.

*GIT STATUS*

Función: Muestra el estado actual del repositorio, incluyendo archivos modificados, archivos en el área de preparación y archivos no rastreados.

*GIT LOG*

Función: Muestra el historial de commits en el repositorio, incluyendo información como el ID del commit, el autor, la fecha y el mensaje del commit.

*GIT DIFF*

Función: Muestra las diferencias entre los archivos modificados y el último commit. Útil para ver qué cambios se han realizado antes de añadir o confirmar los cambios.

*GIT BRANCH*

Función: Muestra una lista de todas las ramas en el repositorio. Indica la rama actual con un asterisco (*).

* GIT BRANCH* < nombre >

Función: Crea una nueva rama con el nombre especificado.

*GIT CHECKOUT* < nombre >

Función: Cambia a la rama especificada. También se puede usar para revertir cambios en archivos.

*GIT MERGE* < rama >

Función: Combina los cambios de la rama especificada en la rama actual. Se usa para integrar cambios de diferentes ramas.

*GIT PULL*

Función: Actualiza el repositorio local con los cambios del repositorio remoto. Ejecuta git fetch seguido de git merge para traer y combinar los cambios.

*GIT PUSH*

Función: Envía los commits locales al repositorio remoto. Actualiza la rama remota con los cambios realizados localmente.

*GIT REMOTE -V*

Función: Muestra las URL de los repositorios remotos configurados para el repositorio local. Las URL se usan para operaciones de fetch, push, etc.

*GIT RM* < archivo >

Función: Elimina un archivo del repositorio y lo marca para ser eliminado en el próximo commit. También se puede usar con la opción --cached para eliminar el archivo solo del área de preparación.

*GIT RESET*

Función: Revertir el área de preparación y/o el repositorio a un estado anterior. Puede ser usado para deshacer cambios no deseados.

*GIT TAG* < nombre >

Función: Crea una etiqueta (tag) en el commit actual. Las etiquetas se usan comúnmente para marcar versiones de lanzamiento.
